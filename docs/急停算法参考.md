# 1. Introduction

## 1.1 Background

Counter-Strike 系列游戏中，玩家为了获得最高的第一枪精度，需要在移动过程中完成 Counter-Strafe（反向急停）。

传统意义上的 Counter-Strafe 判断依赖：

- 玩家当前位置
- 当前速度（Velocity）
- Acceleration
- Weapon Accuracy
- 游戏 Tick
- Server Simulation

然而，CS Hero 不读取游戏内存，不 Hook，不使用 Demo，也无法获取玩家真实速度。

因此，本项目采用 **Input Analysis（输入行为分析）** 的方式，对玩家输入质量进行评估。

需要强调：

**本系统不会判断玩家是否已经真正停止移动。**

本系统仅判断：

> 玩家输入是否符合优秀 Counter-Strafe 玩家所采用的输入模式。

因此所有评分均属于：

Input Quality

而不是：

Movement Accuracy

---

# 1.2 Design Goal

目标：

仅利用：

- Keyboard
- Mouse

分析：

- Counter-Strafe
- Movement Habit
- Input Rhythm
- Fire Timing

不依赖：

- Memory
- Hook
- Demo

- Position

因此本系统具有：

✔ 零侵入

✔ 全平台

✔ 不读取游戏数据

✔ 不影响 VAC

# Final Notes

CS Hero 的核心定位应当始终保持一致：

它是一款 **Movement Input Analyzer**。

它分析的是：

- 输入质量（Input Quality）
- 输入稳定性（Consistency）
- 输入节奏（Rhythm）
- 输入时机（Timing）
- 输入习惯（Habit）

而不是：

- 游戏内部速度（Velocity）
- 武器精度（Accuracy）
- 第一枪命中率（First Shot Accuracy）
- 急停是否真正成功（Counter-Strafe Success）

这是整个项目最重要的边界，也是保证分析结果科学、准确且不误导用户的基础。

当未来接入 Demo、游戏遥测或其他可信数据源时，可以在现有架构之上增加 **Physics Layer**，利用真实速度、武器参数和 Subtick 信息进一步提升分析精度，而无需推翻 Input Analyzer 的设计。
